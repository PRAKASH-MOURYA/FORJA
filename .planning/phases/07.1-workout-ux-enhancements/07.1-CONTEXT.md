# Phase 07.1: workout-ux-enhancements - Context

**Gathered:** 2026-03-23
**Status:** Ready for planning

<domain>
## Phase Boundary

Enhances the workout UX with: a day-by-day custom split builder, back-arrow session guard, exercise history bottom sheet, pre-workout previous-session data inline on TodayScreen, readiness score by muscle group (Recovery Heatmap), and two new home screen cards (PR to Beat, Recovery Heatmap). No new screens outside these features; navigation stays on existing routes.

</domain>

<decisions>
## Implementation Decisions

### Custom Split Builder
- Day-by-day builder: user sets number of days, names each day (e.g., "Push A"), picks exercises from the existing exercise library, configures sets/reps per exercise
- Replaces the currently assigned program (no side-by-side toggle) — old program gone, re-selectable from onboarding templates
- Entry point: Profile screen, under "Current Program" section
- Weight auto-progression after each session driven by existing AdaptiveEngine (recovery + readiness) — this is already in place via Phase 6

### Recovery Heatmap
- 2-column color-coded muscle grid using ForjaCard style — muscle groups: Chest, Back, Legs, Shoulders, Arms, Core
- Recovery color driven by: days since muscle last trained (from SetLogs) + soreness from last check-in combined
- Colors: green (ready), yellow (mild fatigue), red (sore/recovering) — map to existing AppColors.accent / AppColors.warm / AppColors.coral
- Collapsed by default on TodayScreen — shows a summary row (e.g., "3 ready · 2 sore") with expand button to reveal full grid

### Home Screen Cards (TodayScreen layout)
- Both new cards placed **below the exercise list, above the Start Workout button** (existing readiness/WHY banners stay at top)
- Final layout: header → pills → readiness banner → WHY banner → EXERCISES section → ExerciseRows → PR to Beat Card → Recovery Heatmap (collapsed) → Start Workout button
- PR to Beat Card: shows the single top PR opportunity from today's session (exercise name + current PR weight + target weight to beat)

### Pre-workout Data (ExerciseRow)
- Each ExerciseRow on TodayScreen shows last session weight + reps as a subtitle inline (e.g., "Last: 80kg × 8")
- Data pulled from SetLog history via existing workout repository

### Session Guard
- Back-arrow mid-workout triggers a bottom sheet (not a dialog) with two options: "Resume Workout" and "Discard & Exit"
- Consistent with app's existing modal pattern (showModalBottomSheet, transparent background, ForjaCard style)

### Exercise History
- Accessible via bottom sheet when tapping the exercise name inside the WorkoutScreen
- Shows last 3 sessions for that exercise: date + sets summary (e.g., "80kg × 8, 82kg × 6, 77kg × 8")
- Uses existing SetLog data from workout repository

### Claude's Discretion
- Exact grid layout dimensions for the Recovery Heatmap (2-col vs adaptive)
- Bottom sheet height and animation for session guard and exercise history
- Loading/empty states for PR to Beat Card (no prior PRs)
- Exact summary text for heatmap collapsed state

</decisions>

<code_context>
## Existing Code Insights

### Reusable Assets
- `ForjaCard`: use for PR to Beat Card, Recovery Heatmap muscle tiles, session guard bottom sheet body, exercise history bottom sheet body
- `ForjaButton`: Start Workout CTA and bottom sheet actions (Resume / Discard)
- `ForjaPill`: status pills on TodayScreen (already in use)
- `ExerciseRow` (`lib/shared/widgets/exercise_row.dart`): extend to show "Last: Xkg × Y" subtitle inline
- `AppColors` (accent/warm/coral + their Dim variants): directly map to green/yellow/red recovery zones — no new colors needed
- `AppTextStyles` (body, bodyStrong, caption, labelUppercase): all needed text hierarchy is covered

### Established Patterns
- Riverpod `ConsumerWidget` / `ConsumerStatefulWidget` for all screens and providers
- Hive repositories for local data (SetLog, WorkoutLog, CheckIn) — exercise history reads from `WorkoutRepository`
- `showModalBottomSheet` with `isScrollControlled: true, backgroundColor: Colors.transparent` — used in exercise_demo_sheet.dart, same pattern for session guard + exercise history sheet
- `CustomScrollView` with `SliverPadding` / `SliverList` in TodayScreen and HistoryScreen — new cards slot in as `SliverToBoxAdapter`
- Adaptive engine already computes readiness by zone (green/yellow/red) from check-ins — Recovery Heatmap extends this to per-muscle-group granularity

### Integration Points
- `TodayScreen` (`lib/features/today/today_screen.dart`): add ExerciseRow subtitle prop + two new Sliver cards below exercise list
- `WorkoutScreen` (`lib/features/workout/workout_screen.dart`): intercept system back, show session guard bottom sheet, add exercise name tap → history sheet
- `ProfileScreen` (`lib/features/profile/profile_screen.dart`): add "Edit / Create Split" button under current program section
- `WorkoutRepository` / `PRRepository`: source for exercise history and PR data
- `CalculationService`: extend (or new `MuscleRecoveryService`) to compute per-muscle recovery scores from SetLog + CheckIn data

</code_context>

<specifics>
## Specific Ideas

- Recovery Heatmap muscle grid should use existing `AppColors.accentGlow` / `AppColors.warmDim` / `AppColors.coralDim` as background tints on each tile — matches readiness banner pattern
- PR to Beat Card target weight: current PR + smallest standard increment (2.5 kg) — same increment logic used in workout screen weight stepper
- Collapsed Heatmap summary row should visually match the existing WHY banner container style (bgElevated, border, rounded corners)
- Exercise history sheet: show 3 rows max, date in caption style, weights in bodyStrong accent color

</specifics>

<deferred>
## Deferred Ideas

- Sleep-based weight progression (HRV, sleep hours → adjust target weight) — Phase 8 (wearables/HealthKit integration)
- Dedicated full exercise detail screen with history chart (trend line over time) — potential Phase 8 or future enhancement
- Muscle recovery notifications ("Chest fully recovered — ready to train!") — future retention feature

</deferred>
