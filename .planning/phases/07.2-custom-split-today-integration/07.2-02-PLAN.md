<objective>
Fix WorkoutScreen to use the exercises passed by TodayScreen instead of the hardcoded _pushDayExercises list.

Root cause: WorkoutScreen has a hardcoded _pushDayExercises list and hardcoded 'Push Day' title. TodayScreen does call context.push('/workout', extra: sessionExercises) but the router doesn't forward this extra to WorkoutScreen — it constructs `const WorkoutScreen()` with no parameters. WorkoutScreen then ignores the passed exercises entirely and starts the workout with the hardcoded list.

Fix: (1) TodayScreen passes a Map {exercises, dayName} via extra. (2) Router reads extra and constructs WorkoutScreen(exercises: ..., dayName: ...). (3) WorkoutScreen accepts and uses these in _startWorkout and _initSetState.
</objective>
